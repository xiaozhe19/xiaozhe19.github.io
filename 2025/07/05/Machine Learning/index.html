Linear Regression Model

  • Cost Function

    • Squared error cost function:
      • Linear regression is to find a pair of w, b to make J small enough.

  • Gradient Descent - Find the best w, b

    • minimize any function

    • $w = w-\alpha \frac{\partial}{\partial w}J(w,b)$

      • $b = b-\alpha \frac{\partial}{\partial w}J(w,b)$

      • $\alpha $ Learning Rate 0-1 control how big one step you go

      • dw = (1/m) * np.sum(error * x)

        db = (1/m) * np.sum(error)

    • repeat until convergence（收敛）

    • update w and b at same time

      • temp_w = w-A temp_B=b-A w = temp_w b = temp_b

    • there will be more than one local minimum

      • but for linear regression -> only one

    • Learning rate is not always better to make it smaller.

      • too small: cant find minimum point within limited steps

  • Multiple Feature

    • $\vec{x}^{(i)}$ features of $i^{th}$ training example

    • Model

      • $f_{w,b}(x) = w_1x_1+w_2x_2+w_3x_3+b$
      • $\vec{w}=[w1,w2,w3,…,wn]$
      • $\vec{x} = [x1,x2,x3,…,xn]$
      • $ f_{\vec{w},b}=\vec{w}\cdot\vec{x}+b$

    • Vectorization

      • makes code shorter and faster
      • np.dot(x,y) 向量 x, y 的点积

    • Feature Scaling

  • Normal equation

    • only for linear regression
    • solve for w, b without iterations(one time to solve)
    • but when number of feature is above 10 000, it will be very slow

  • Feature Scaling

    • make a range that is comparable, make gradient descent faster.

    • method

      • divide a number to make features in a same range

      • mean normalization 均值归一化

        • find x1’s average -> $\mu_1$
          • $x_1 = \frac{x_1-\mu_1}{max(x1)-min(x1)}$

      • Z-Score normalization

        • find standard deviation $\sigma$
        • $x_1=\frac{x_1-\mu_1}{\sigma}$

    • aim for a range of -1 to 1 for each feature

  • Choose Learning Rate

    • learning curve

    • Automatic Convergence Test

      • let a $\epsilon$ be 0.001
      • if J decrease less than $\epsilon$ , you can declare J is convergence
      • difficult to choose a right epsilon

    • chose a very small number -> debugging step

    • draw J and iterations curve to see where to convergence

  • Feature engineering

    • combining original features to define some new feature to make model easy to train

Logistic Regression

  • sigmoid function
    • $g(z) = \frac{1}{1+e^{-z}}$ possibility to output 1
    • $z = \vec{w}\cdot\vec{x}+b$ and pass z into g(z)
    • this is $f_{\vec{w},b}(\vec{x})$, shows the possibility to be positive
  • decision boundary
    • $z = \vec{w}\cdot\vec{x}+b=0$
  • Cost Function
    • Squared error cost is a non-convex function so we should use another type of function
    • Logistic Loss Function
    • Simplified version
      • $L(f_{\vec{w},b}(\vec{x}^{(i)}),y^{(i)}) = -y^{(i)}log(f_{\vec{w},b}(x^{(i)}))-(1-y^{(i)})log(1-f_{\vec{w},b}(x^{(i)}))$
    • $dw = \frac{1}{m}\vec{x}\cdot(\vec{y_{hat}}-\vec{y})$
    • $db = \frac{1}{m}(\vec{y_{hat}}-\vec{y})$

      Demand Prediction

  • General method of front propogration

    • def dense(a_in,W,b, g): # This is an example of using loop
    units =  W.shape[1]
    a_out = np.zeros(units)
    for j in range(units):
        w = W[:,j] #Pull out column j of a matrix
        z = np.dot(w,a_in)+b[j]
        a_out[j] = g(z) #sigmoid function
       return a_out
def sequential(x):
    a1 = dense(x, W1, b1)
    a2 = dense(a1, W2, b2)
    a3 = dense(a2, W3, b3)
    a4 = dense(a3, W4, b4)
    f_x = a4
    return f_x

Alternative for Sigmoid Function

  • for the non-binary situation
  • another useful function ReLU
    • if z <0 -> g(z) = 0
    • if z >= 0 -> g(z) = z
    • g(z) = max(0, z)
  • Choose Activation Function
    • Output Layer
    • Binary Classification -> sigmoid function
    • Regression problem -> linear
    • non-negative result -> ReLU
    • Hidden Layers
      • ReLU most used
        • ReLU is faster
        • ReLU only goes flat in one direction
        • Flat will make gradient descent very slow
  • use activation function in all the node will turn neural network into a linear regression
  • use all sigmoid function is to logistic regression
  • So we need ReLU to decrease the linear feature of the model to better fit the statics

Multiclass

  • Softmax Regression
    • distinguish z1 z2 z3 z4
    • $z_n = \vec{w_n}\cdot\vec{x}+\vec{b_n}$
    • $a_1 = \frac{e^{z_1}}{e^{z^1}+e^{z^2}+e^{z^3}+e^{z^4}}$ Activation
    • $Loss = L(\vec{a},y) = -log a_n if y = n$
    • Number Roundoff Errors
      • computer only have a limited storage to save numbers
      • so some accurate number will be roundoff to make a wrong answer
      • so also our way to calculate softmax is right, there will be still some wrong answer
      • solution
        • in Tensorflow we can calculate loss function in one time， not predicted function first, then put the answer into loss function.
        • model.compile(loss = SparseCrossEntropy(from_logistic = True)) replace model.compile(loss = SparseCatagoricalCrossEntry()) this new command contains softmax regression as output layer, so use linear as output
    • Multi-Classes and Multi-Labels
      • Multi-Classes: every example can only belong to one class
      • Multi-Labels: every example can have many labels
